iOS和android平台上的差异

（1）小心使用modal

项目开发中，发现在iOS平台上，如果一个界面上同时显示两个及以上的modal组件，界面会卡死无法进行任何的交互操作。

解决办法：页面开发中如果需要使用到浮层，尽量避免使用Modal组件避免同一界面会同时出现两个Modal浮层的情况，或者使用三方库图层组件来代替，例如teaset中的Overlay。

（2）Text组件文字居中

Text组件中的textAlignVertical只在Android上有效，iOS上无效，设置该属性文字内容无法在Text组件中垂直居中，文字居中可使用View包裹Text来实现，封装后屏蔽双端上的差异。

（3）视频播放ios不暂停循环播放

备注：this.setState({…},()=>{})，设置状态函数中有一个回调函数，会在页面刷新渲染完成后回调。
进度初始化需要放在pause状态后设置。

（4）输入法问题

点击输入框，键盘弹起在android和ios的效果不一样

使用Webview的闪退问题

Android上react-native-webview组件对返回键事件的处理和本身物理按键返回退出页面冲突，库内部闪退，造成程序闪退，重写安卓物理返回按键，并拦截事件屏蔽webview的返回。

日期时间格式化的问题

node.js转化异常，js端可用的new Date（time）.getTime()无法正确格式化时间戳，引入moment库解决

其他问题
